Let's Start with the Basics:

Open your favorite text editor (hopefully Atom), and create a file with a .html file extension. I'd suggest index.html, but only extension is important. It allows for your editor to use syntax highlighting (so everything isn't black and white) and for your computer to open it using a web browser.

There are two major portions of a html file: the head, and the body. The head is just meta data about your website. This includes the title of your page, and external scripts and stylesheets. Additionally, you should always, always add <!DOCTYPE html> to the top of your file, before anything else. This is a declaration of what version of HTML the file was written in. In HTML5, the current version of HTML, this is the only one.

HTML revolves around tags, denoted by <>. You have alreay inspected plenty of tags at this points, and it is easy to see this. Even this text is encapsulated in a tag. There are many, many, tags, but most of the basics can be accomplished using the follwing.

HTML Element Description
<!DOCTYPE> Defines the version of HTML used. No need for a closing tag.
<html> Defines an HTML document.
<title> The title of the current page, declared in the head. Displayed on the top of the tab.
<body> What is displayed by the ebpage. Text, images, etc.
<h#> A heading, the bigger the #, the smaller the heading
<p> Basic text. It does not have to be four sentences of more.
<table>, <th>, <tr>, <td> Different parts of a table, representing the table itself, the table header, table row, and table data, respectively. Use this table as an example of how to use them together.
<a href="#"> A hyperlink to another webpage. Use pound to make a "dead link."
<img src=""> Indicates a picture, with the src being the location of the picture.
<ul> An unordered list. Use with <li> to make a bulleted list.
<ol> An ordered list. Use with <li> to make a numbered list.
<li> An element in a list.
<div> Defines a container. It implies all other HTML elements in this container are related.
<span> Denotes that this text should be inline with surrounding elements. It allows you to style certain things differently.
<nav> Indicates a list of links. This is an accessibilty option.
<pre> Means that the following text is already preformatted, thus any whitespace is conserved. This is mostly for placing code sampkes into HTNL.
